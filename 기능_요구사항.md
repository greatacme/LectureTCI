# LectureTCI 기능 요구사항

LectureTCI는 참여형 강의를 위한 웹 화면입니다. 아래 항목에 생각나는 내용을 자유롭게 적어주세요. 비어 있어도 괜찮고, 문장/키워드/예시 화면 흐름 아무 형태나 좋습니다.

## 1\. 프로젝트 개요

* 한 줄 설명: 수강생들이 TCI 검사 예시를 체험하게 하는 웹페이지
* 주 사용자: 강사와 수강생
* 사용할 상황: TCI 검사에 대한 강의
* 가장 중요한 목표: 다수 사용자가 동시에 접속해서 사용
* TCI검사는 MBTI 검사와 유사하다고 생각하자

## 2\. 강의 진행자 기능

* 강의 시작/종료: 수강생용 체험 페이지의 시작과 종료를 제어한다.
* 질문 또는 활동 생성: 
	- 사전에 검사항목 예제를 선정하고 항목별 점수산정 기준을 산정한다.
	- 참여자들이 입력을 마치면 결과를 취합하고 개인별 결과 보기 화면을 오픈한다.
	- 강의를 마치면 참여 페이지를 종료하고 자료를 삭제한다.

* 참여자 응답 확인:
* 실시간 피드백:
* 결과 저장/내보내기: 수강생이 입력한 내용과 결과는 저장하지 않고, 해당 강의의 체험이 종료되면 폐기된다.
* 기타:

## 3\. 참여자 기능

* 접속 방식: 공유된 URL로 접속하고, 임시로 사용할 ID를 입력한다.
* 응답 방식: TCI 검사항목별로 응답을 입력한다.
* 익명/실명 여부: 임시 사용할 ID가 배부된다. 예를들어 휴대폰 8자리 혹은 메일주소 등
* 참여 상태 표시:
* 기타:

## 5\. 화면 구성

* 진행자 화면: 
	- TCI 검사항목 관리 : 데이터베이스로 관리
		- 검사SET 입력/수정/삭제
		- 검사문항 입력/수정/삭제
		- 검사문항은 검사SET별로 별도 관리된다

	- 체험 진행 관리
		- 체험 시작 버튼 : 검사SET를 선택하고 체험시작을 하면 수강생을 위한 URL이 생성된다
		- URL 복사 버튼 : 클립보드에 복사되고 메신저 등을 통해 수강생에게 공유된다
		- 참여자 입력 현황 조회 : 참여자의 ID와 검사항목 입력 진행 상태(시작 전/입력 중/입력 완료)를 볼 수 있다.
		- 입력 종료 버튼 : 참여자의 검사항목 입력을 중단시킨다.
		- 결과 취합 버튼 : 입력된 검사항목 응답으로 전체 점수를 취합하고 개인별 결과를 생성한다.
		- 결과 공유 버튼 : 참여자 화면에 결과보기 버튼을 활성화 시킨다

* 참여자 화면:
	- ID 입력 창 : ID입력을 완료하면 검사 화면으로 이동한다.
	- 검사 화면 : 검사항목이 표시되고 응답을 입력할 수 있다. 입력이 종료되면 완료버튼을 누른다.
	- 결과보기 버튼 : 버튼을 누르면 결과 화면으로 이동한다.
	- 결과 화면 : 검사 결과를 조회할 수 있다.
